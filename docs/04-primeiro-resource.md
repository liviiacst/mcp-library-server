# O Primeiro Resource

Um Resource representa uma informação disponibilizada pelo servidor MCP.

Ao contrário de uma Tool, um Resource não executa ações nem altera o estado da aplicação.

Neste projeto criamos o Resource `library://authors`, responsável por disponibilizar a lista de autores cadastrados.

## O que aprendi

- Resources representam informações.
- Tools representam ações.
- Um Resource possui uma URI.
- Um Resource não modifica dados.
